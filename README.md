Stroke Risk Prediction using Machine Learning and Deep Learning

This repository contains the official implementation of four models for stroke risk prediction using heterogeneous electronic health records (EHRs), as described in our paper:  
"Neuro-Tree Fusion: A Hierarchical Sparse-recurrent Transformer for Stroke Risk Prediction Using Heterogeneous Tabular Data".

The four models implemented are:

1. XGBoost – Gradient boosted decision trees (traditional ML)  
2. LightGBM – Gradient boosted decision trees with leaf-wise growth  
3. TabNet – Attentive interpretable tabular learning (deep learning)  
4. HSRT (Hierarchical Sparse-Recurrent Transformer) – Our proposed architecture combining differentiable hard selection, permutation-invariant encoding, and sparse-aware attention

All models are evaluated under the same EasyEnsemble framework to handle severe class imbalance, with temporal validation (train: 2000–2016, validation: 2017, test: 2018) on a large-scale dataset from 200 healthcare institutions in Southern Sichuan, China.

📊 Data Preparation

The models expect pre-processed embedding data generated by `data_loader.py`. The raw data files (CSV format) should be placed in the project root with the following naming convention:
- `healthy16.csv`, `healthy17.csv`, `healthy18.csv` – healthy control samples
- `Stroke16.csv`, `Stroke17.csv`, `Stroke18.csv` – stroke patient samples

Each file must contain a `label` column (0 = healthy, 1 = stroke). The `data_loader.py` performs type-aware encoding, missing value handling, and creates balanced subsets for EasyEnsemble.

If you wish to use your own data, modify the file paths in each model script accordingly.

🚀 Usage

Each model script is self-contained and can be run independently. By default, they execute 3 independent runs (with different random seeds) and report mean ± std metrics.

1. XGBoost
```bash
python xgboost_model.py
```
Outputs: `xgboost_easyensemble_output_run_*/` with models, metrics, feature importance plots, and ROC curves.

2. LightGBM
```bash
python lightgbm_model.py
```
Outputs: `lightgbm_easyensemble_output_run_*/` with similar structure.

3. TabNet
```bash
python tabnet_model.py
```
Outputs: `tabnet_easyensemble_output_run_*/` – includes model checkpoints, attention masks, and feature importance.

4. HSRT (proposed model)
```bash
python hsrt_model.py
```
Outputs: `hsrt_easyensemble_output_run_*/` – includes attention heatmaps, field importance rankings, and DSS mask visualizations.

After all runs, the script aggregates results and saves a summary in `./output/summary.json`.

📈 Key Features of Each Model

Shared Components
- EasyEnsemble: 10 sub-models are trained on balanced subsets (undersampled majority class) and combined via max‑voting or weighted averaging.
- Temporal split: Training (2000–2016), validation (2017), test (2018) – mimics real-world deployment.
- Evaluation metrics: Accuracy, Precision, Recall (Sensitivity), Specificity, F1-score, AUC, Balanced Accuracy, G-mean.
- Visualization: ROC curves, confusion matrices, feature/field importance bar charts.

Model-Specific Highlights

| Model | Key Characteristics |
|-------|----------------------|
| XGBoost | Gain-based feature importance, early stopping |
| LightGBM | Leaf-wise growth, categorical feature support, fast training |
| TabNet | Sequential attention for feature selection, Sparsemax, interpretable masks |
| HSRT | Gumbel-Softmax hard selection, permutation-invariant encoding, sparse-aware multi-head attention, state truncation |
